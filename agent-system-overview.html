Section 1
Agent Workflow — Step-by-Step
1
Scope & Context Definition
Orchestrator
The market-researcher orchestrator sets the research scope (Australian bedding, mattresses, sleep products, bedroom furniture) and defines the watchlist of 13 competitors. It invokes the lkg-furniture-market-digest skill for context framing and source-mix rules.
market-researcher lkg-furniture-market-digest skill
2
Sector Overview
Subagent
Delegates to sector-overview-agent. Drafts market structure, value chain, demand drivers, supply-chain signals, and why-now narrative. Uses web_search → stable URL fallback → synthetic fallback (labelled). Returns findings with source citations and review flags.
sector-overview-agent sector-overview skill web_search · web_fetch
3
Competitive Landscape
Subagent
Delegates to competitive-analysis-agent. Maps competitor moves — promotions, store changes, range updates, online signals, investor commentary. Each signal includes: what happened, source, why it matters, GM/Board relevance, confidence, and human-review flag.
competitive-analysis-agent competitive-analysis skill web_search · web_fetch
4
Peer Comps & Operating Signals
Subagent
Delegates to comps-analysis-agent. Builds comparable operating-signal benchmarks for public furniture/home-retail peers (sales growth, margins, inventory, online sales, promotional intensity). Preserves spreadsheet discipline: consistent peer set, formulas, source hierarchy.
comps-analysis-agent comps-analysis skill web_search · web_fetch
5
Ideas Shortlist
Subagent
Delegates to idea-generation-agent. Generates evidence-backed GM and Board follow-up actions (promotion response, range review, inventory check, board watch items). Each idea has: evidence, why it matters, suggested audience, risk-if-ignored, recommended next step.
idea-generation-agent idea-generation skill
6
GM / Board Classification
Subagent
Delegates to gm-board-classifier-agent. Routes every material signal as GM (operational/tactical), Board (strategic/risk), Both, or Ignore. Classification is advisory — no distribution without human approval.
gm-board-classifier-agent gm-board-classifier skill
7
Human Review Gate
Human-in-the-Loop Mandatory
The orchestrator presents a classified draft and pauses for human approval. The reviewer controls: item approval, classification overrides, edits to interpretation, suppression of weak items, and the final circulation decision. No output is generated or sent until this gate is cleared.
Human Reviewer No auto-distribute
8
Word Output Generation
Output
Delegates to note-writer-agent. Assembles approved content into three Microsoft Word artifacts: GM Weekly Digest, Board Weekly Digest, and Internal Source Log. Optionally hands off to pptx-author for slide output. Files written to output/ folder.
note-writer-agent pptx-author skill (optional) Word · PPTX output

  • Harvey Norman · Domayne
  • IKEA Australia

🧪 Synthetic Demo Fallback (Last Resort)
  • Only when live search & stable URLs fail
  • Requires explicit demo/fallback context
  • Always labelled SYNTHETIC DEMO FALLBACK
  • Excluded from GM/Board routing without human approval
  • Separated in source log under its own section
🗄️ Session Memory & Outputs
  • No persistent cross-session memory (by design)
  • Source logs stored per run in output/
  • GM Digest: lkg-furniture-gm-weekly-digest-{DD-MM-YYYY}.docx
  • Board Digest: lkg-furniture-board-weekly-digest-{DD-MM-YYYY}.docx
  • Source Log: lkg-furniture-internal-source-log-{DD-MM-YYYY}.docx
  • Run logs & cost logs planned for production
🚫 Excluded Data
  • No confidential LK Group / Hypnos data
  • No employer or private client information
  • No CapIQ / FactSet in cookbook mode (future)
  • No auto-distribution to GM or Board
  • No unsourced claims passed as facts
Section 4
Governance & Cost Controls
🛡️ Governance Rules
  • ✅Every material item must cite a source URL
  • ✅Facts and interpretation are always separated
  • ✅Low-confidence / high-impact items flagged for review
  • ✅GM/Board routing is advisory — never automatic
  • ✅Human approval required before any circulation
  • ✅Unsourced items marked [UNSOURCED]
  • ✅All outputs include a draft-only disclaimer
  • ✅Source log included in every output package
💰 Cost Controls
  • ⚡Limit source scan to 10 relevant sources by default
  • ⚡Prefer primary sources over repeated broad searches
  • ⚡Cache source logs where possible between runs
  • ⚡Record API & search costs for operational planning
  • ⚡Stable URL fallback reduces redundant search calls
  • ⚡Synthetic fallback only as absolute last resort
  • ⚡Claude Sonnet 4.6 used across all agents (cost-balanced)

Future Development Roadmap
Phase 1 — Production Data Connectors
Replace web_search with governed, scheduled connectors
📢
ASX Announcements Connector
Automated ingestion of ASX market announcements for furniture/retail peers. Replaces manual web_fetch of ASX search page.
📉
ABS Retail Trade Connector
Scheduled pull of ABS retail trade and household-goods data for demand context, replacing manual URL fetch.
🏢
Company Investor Pages Monitor
Continuous monitoring of investor relation pages (Adairs, Nick Scali, Temple & Webster, Harvey Norman) for new releases.
🕵️
Competitor Web Monitor
Structured scraping of competitor product pages, promotion banners, and pricing to detect range/price changes automatically.
Phase 2 — Premium Data & Analysis
Institutional-grade financial data integration
💼
CapIQ MCP Integration
Enable Capital IQ MCP connector for the comps-analysis-agent to pull structured financials, valuation multiples, and consensus estimates.
📊
FactSet MCP Integration
Enable FactSet MCP for real-time market data, sector indices, and peer benchmarking in comps and sector-overview subagents.
🧮
Excel/Spreadsheet Output
Extend comps-analysis-agent to produce structured Excel outputs with formulas, peer comparison tables, and statistical summaries.
📐
Valuation Multiples Benchmarking
Add EV/EBITDA, P/E, and FCF yield benchmarking for public peers once CapIQ/FactSet connectors are live.
Phase 3 — Output & Workflow Automation
Governed distribution and scheduling
📄
Microsoft 365 Output Connector
Push approved Word/PPTX artifacts directly to SharePoint, OneDrive, or Teams channels via Microsoft Graph API.
✔️
Approval Workflow Connector
Formalize human review with a structured approval interface — item-level approve/reject/edit, audit trail, and approval state tracking.
⏰
Scheduled Runs (Cron)
Weekly scheduled trigger via Anthropic Managed Agent platform — auto-generate draft digest, alert reviewer, await approval before output.
🔔
Failure Alerts & Run Monitoring
Failure notifications, run logs, model/tool cost logs, and output versioning for operational governance in production.
Phase 4 — Intelligence & Personalisation
Learning from feedback, expanding coverage
🧠
Cross-Run Memory & Source Caching
Persist source logs and prior signals across runs. Enable delta detection — highlight only new or changed signals vs. last digest.
🎯
Reviewer Preference Learning
Track human approval/rejection patterns to improve GM/Board classification accuracy over time without retraining the model.
🌏
Expanded Market Coverage
Extend beyond AU to NZ and UK markets where Hypnos Group has presence. Add sleep-technology and wellness adjacency tracking.
🤖
Agentic Skill Evals
Run systematic evals on each subagent's skill output quality — source reliability, classification accuracy, idea relevance scoring.
